This is a Bun runtime project that replaces Node.js. as the default runtime for Bun. i still will use node.js for if in future the bun doesnt support a library im using or smth like that,

I want to use bun and bunx as the default runtime for Bun.

## Primary Object of this Project

-   Serve as the primary backend for Haizra an Application whos frontend is built with Next.js and backend with Expo React Native, The backend will Perform the following tasks

*   Make API calls for front end
*   Document those API calls with Swagger and serve it on /api-docs
*   Make Database connections and perform database operations
*   Make Authentication and Authorization for the application
*   Make Payment Gateway for the application
*   Using Bun and refrerring to `https://bun.com/docs` as the primary documentation for Bun
*   Make Models and Migrations for the database
*   Make API calls for the application

## Coding Standards to follow

-   Use TypeScript for the application
-   Use Bun and Bunx for the application
-   Use Bun's default runtime for Bun
-   Each API should have a description and tags
-   Follow modern API Standards document them Properly with Swagger
-   Use proper error handling and logging
-   Use proper versioning for the API
-   Use proper authentication and authorization for the API
-   Use proper database design and normalization
-   Use proper database indexing and optimization
-   Use proper database backup and recovery
-   Use proper database security and encryption
-   Use proper database performance and scalability
-   Use proper database monitoring and logging
-   Use proper database documentation and maintenance
-   Use Proper Type Safety while making models
